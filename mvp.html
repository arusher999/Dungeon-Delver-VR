Core Idea Demonstration

The MVP for Dungeon Delver VR focuses on delivering a testable, immersive VR experience accessible via the Quest Pro. Our core features include a working VR dungeon environment, basic player movement, interaction, combat system, and an inventory system.

  • Fully publish the game through Roblox Studio for public testing.
  • A working VR dungeon environment.
  • Basic player movement and interaction.
  • Combat system with enemies.
  • Inventory system (pickup/store/switch items).

Must Be FULLY WORKING by Week 11

  • Stable movement & interaction mechanics.
  • Combat mechanics (basic AI, attacking, player health system).
  • At least one functional puzzle or trap.
  • Basic UI elements for health, inventory, and player feedback.

Current Feature Priorities

Essential:

  • VR movement and interaction (teleportation/smooth locomotion).
  • Basic combat mechanics (melee/simplified ranged).
  • Enemy AI with pathfinding & basic attacks.
  • Functional inventory system.

Nice-to-Have (Future Plans):

  • Puzzle mechanics (levers, pressure plates, etc.).
  • Haptic feedback for combat.
  • Spatial audio for immersion.

Extra:

  • Procedural dungeon generation – VERY DIFFICULT FOR ROBLOX PLATFORM.
  • Adaptive difficulty AI – challenging but possible.
  • Advanced enemy attack patterns.

Tech/Design Review

  • Scripting: Ensure Lua scripts handling VR interactions are optimized.
  • UI/UX: Prioritize intuitive controls, clear visual feedback, and smooth movement.
  • Performance: Avoid rendering overload—optimize environment assets.

Initial Implementation Plan

  • Hamiz: Finalizing environment & integrating combat mechanics, and publishing the game to the Roblox Platform.
  • Aarush: Focus on testing VR live and debugging platform.
